Design Concepts


Initial Sketches

We started our design task by asking ourselves the following question: How can we remind the user that they're getting distracted without being too intrusive? This led us to create the 10 sketches below, where we attempted to create a layer between the user and the distracting website. A few sketches of the settings page of the Chrome Extension were also attempted to be thorough with the user's interface.


Previous Next


Elaborated Sketches

Then we settled on the three sketches you can see below, mainly because they were the most likely to intrigue the user due to self-education potential such as learning a new language or typing faster, or simply by adding a emotional aspect, such as a virtual pet. We agreed that these sketches provided just enough incentive for the user to want to use the application, while at the same time not irritating the user too much.

Prototypes

Facebook background

For our extension's background, we created a Facebook template, so that we can overlay our extension prototypes on top of it.



Prototype 1

Previous Next

This first prototype is simple and easy to understand. To access the website, the user has to type in the sentence is displayed on the page. Every sentence is randomly generated so it never becomes redundant. Once typed in, a “continue” button appears so the user can click to access the website.



Prototype 2

Previous Next

The second prototype has an educational purpose. To access the website, the user has to answer correctly the question that is being displayed. On a correct answer, three choices appear on the screen: learn more, close tab and continue to website. On a wrong answer, the right answer is revealed but the user has to try again. Here you can also see the settings popup that shows up when you press the settings button in Chrome.



Prototype 3

Previous Next

The third prototype wants to be fun. Each user has its own pet. Every time the user wants to access a website, he has the choice to either feed the pet with time, or to skip feeding to get to the website. Whenever the user skips feeding, the pet loses health and eventually dies. Once the pet gets fed enough times, it will level up and grow.



Usability Goals

1. The task required before accessing the blocked website is feasible

Fundamentally, the task required to access the website can’t be impossible to solve. Everyone should be able to solve it given enough time. Otherwise, this application simply becomes a website blocker, which is not our intended.


2. The user has to be able to easily navigate to the application

It is important for the user to be able to change the settings of the extension to make it more personal, or to change the difficulty level and other settings that would increase user retention. If the application can’t be personalized, or if the user can’t find the settings, they are less likely to use the app in the long run.


3. The task to solve shouldn’t be hard or annoying irritate the user

It is important that the interface with the application isn’t irritating to the user. The user should find that the task is somewhat enjoyable, which makes it worth keeping as a layer between them and their distracting website.


4. The user can easily understand what task they have to do to access their blocked website

The user has to be able to understand what the task is relatively quickly in order for them to solve it. The instructions on the page have to be direct and minimalistic so that the user doesn’t waste too much time understanding the problem before solving it.


5. The task required before accessing the blocked website isn’t too easy

It’s fundamentally important that the task required to access the website isn’t too easy. We want the extension layer to be reasonably effective at stopping the user in his tracks and letting him reflect about their intent to go on the website.
